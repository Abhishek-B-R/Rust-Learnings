## Packages: A Cargo feature that lets you build, test, and share crates
## Crates: A tree of modules that produces a library or executable

# Crate :

## A crate is the smallest amount of code that the Rust compiler considers at a time

### A crate has 2 types: Binary and Library Crate
- **Binary Crate** : Binary crates are programs that can be compiled to an executable file. They must have a main function.
- **Library Crate** : Library crates are code that can be used in other programs. They don't have a main function.

#### Generally **crate** refers to **Library crate**

# Package :

## A package is a bundle of one or more crates that provides a set of functionality. A package contains **Cargo.toml** file which describes how to build those crates.

### A package has 2 types: Binary and Library Package
- **Binary Package** : Binary packages are programs that can be compiled to an executable file. They must have a main function. Main function is located in the src/main.rs file.
- **Library Package** : Library packages are code that can be used in other programs. They don't have a main function. Library package is located in the src/lib.rs file.

#### Generally **package** refers to **Library package**
